Requête pour exemplifier la notion de "big data":

SELECT ?person ?name ?birth_date ?death_date ?gender ?country_of_citizenship ?occupation WHERE { ?person wdt:P31 wd:Q5. ?person rdfs:label ?name. OPTIONAL { ?person wdt:P569 ?birth_date } OPTIONAL { ?person wdt:P570 ?death_date } OPTIONAL { ?person wdt:P21 ?gender } OPTIONAL { ?person wdt:P27 ?country_of_citizenship } OPTIONAL { ?person wdt:P106 ?occupation } FILTER(LANG(?name) = "en") } LIMIT 100

Cette requête récupère les informations relatives à tous les individus (wd:Q5) comme leur nom, date de naissance, date de mort, genre, pays de nationalité et occupation. Elle utilise les propriétés (wdt:) pour récupérer ces informations et filtre pour s'assurer que les résultats soient en anglais. Cela peut retourner un grand nombre de résultats, ce qui illustre la notion de "big data".

Requête pour exemplifier la notion de "high dimensional data":

SELECT ?book ?title ?author ?publisher ?publication_date ?language ?genre ?isbn ?number_of_pages WHERE { ?book wdt:P31 wd:Q571. ?book rdfs:label ?title. OPTIONAL { ?book wdt:P50 ?author } OPTIONAL { ?book wdt:P123 ?publisher } OPTIONAL { ?book wdt:P577 ?publication_date } OPTIONAL { ?book wdt:P407 ?language } OPTIONAL { ?book wdt:P136 ?genre } OPTIONAL { ?book wdt:P212 ?isbn } OPTIONAL { ?book wdt:P1104 ?number_of_pages } FILTER(LANG(?title) = "en") } LIMIT 100

Cette requête récupère les informations relatives à tous les livres (wd:Q571) comme leur titre, auteur, éditeur, date de publication, langue, genre, ISBN et nombre de pages. Elle utilise plusieurs propriétés (wdt:) pour récupérer ces informations et filtre pour s'assurer que les résultats soient en anglais. Cela peut retourner un grand nombre de résultats et chaque livre peut avoir plusieurs propriétés associées, ce qui illustre la notion de "high dimensional data".
